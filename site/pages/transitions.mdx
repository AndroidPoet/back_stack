# Transitions

A destination's transition is just the `Page` you return from `pageBuilder` —
there's no separate transition registry. `TransitionPage` covers the common
cases plus the full Material motion set, all dependency-free.

```dart
pageBuilder: (context, key, pageKey) => switch (key) {
  Step2()  => TransitionPage.sharedAxisHorizontal(key: pageKey, child: screenFor(key)),
  Detail() => TransitionPage.sharedAxisScaled(key: pageKey, child: screenFor(key)),
  Tab()    => TransitionPage.fadeThrough(key: pageKey, child: screenFor(key)),
  Toast()  => TransitionPage.fade(key: pageKey, child: screenFor(key)),
  _        => MaterialPage(key: pageKey, child: screenFor(key)),
};
```

## Pick the motion by the relationship

- **Shared axis X** (`sharedAxisHorizontal`) — a step *forward among peers*
  (a wizard, an onboarding pager).
- **Shared axis Y** (`sharedAxisVertical`) — the vertical sibling.
- **Shared axis Z** (`sharedAxisScaled`) — a step *into a hierarchy* (list → detail).
- **Fade through** (`fadeThrough`) — switching between *unrelated* destinations
  (e.g. bottom-nav tabs), where there's no spatial link to imply.
- Plus `fade`, `slideUp`, `scale`, and `none`.

The shared-axis and fade-through motions animate the *outgoing* screen too, so
both move together the way Material intends.

## A destination can carry its own transition

Mix in `NavPage` and the key builds its own `Page` — then you don't even need a
`pageBuilder`:

```dart
class Toast extends NavKey with NavPage {
  const Toast(this.message);
  final String message;
  @override
  Page<void> buildPage(BuildContext context, Widget child, LocalKey pageKey) =>
      TransitionPage<void>.fade(key: pageKey, child: child);
}
```

## Modals as stack entries

`DialogPage` and `SheetPage` make a dialog or bottom sheet a real route — back-
dismissable and, under the Router, URL-tracked — instead of an imperative
`showDialog` floating outside the stack.
